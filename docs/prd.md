# 需求文档

## 1. 应用概述

- **应用名称**：my-homepage
- **应用描述**：Jesse 的个人主页，包含个人信息展示区与数字分身聊天区，面向抖音观众与潜在合作伙伴，整体风格简约清爽、具有科技感。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 抖音观众：希望了解 Jesse 是谁、在做什么
- 潜在合作伙伴：希望快速了解背景、能力方向及联系方式

### 2.2 核心使用场景
- 访客打开主页，浏览个人基本信息与介绍
- 访客通过数字分身聊天区提问，获取关于 Jesse 的相关回答

---

## 3. 页面结构与核心功能

### 3.1 页面总览

```
my-homepage
└── 首页（单页滚动布局）
    ├── 顶部导航栏
    ├── Hero 区（头像 + 名字 + 一句话介绍）
    ├── 个人信息展示区
    └── 数字分身聊天区
```

### 3.2 顶部导航栏
- 左侧显示站点名称或 Logo 文字：Jesse
- 右侧提供页内锚点导航：关于我 / 数字分身
- 导航栏固定在页面顶部，滚动时保持可见
- 移动端折叠为汉堡菜单，点击展开导航项

### 3.3 Hero 区
- 展示头像（圆形裁切，居中或左对齐）
- 展示名字：Jesse
- 展示一句话介绍：一个正在学习用 AI 做产品的抖音内容策划
- 背景使用深蓝色或紫色渐变，文字使用白色，整体呈现科技风

### 3.4 个人信息展示区
- 锚点 ID：关于我
- 以卡片或分组列表形式展示以下信息：
  - 目前在做：想用 AI 做一些抖音内容
  - 兴趣方向：AI 应用、健身
  - 个人特点：喜欢把复杂的问题简单化
- 信息层次清晰，排版简洁，不堆砌装饰元素

### 3.5 数字分身聊天区
- 锚点 ID：数字分身
- 区域标题：和 Jesse 的数字分身聊聊
- 页面内嵌聊天气泡式 UI，包含：
  - 消息展示区：展示对话历史（用户消息靠右，分身回复靠左）
  - 输入框：单行文本输入，支持回车或点击发送按钮提交
  - 发送按钮
- 数字分身预设知识范围（用于前端 prompt 构建或接口调用）：
  - 职业身份：内容策划
  - 最近在做：搭建自己的主页和作品集
  - 擅长方向：内容表达、AI 应用、知识整理
  - 常见问题参考：
    - 你现在在做什么？
    - 你有哪些作品？
    - 怎么联系你？
- 分身回复内容基于以上预设知识生成，回答风格简洁、自然
- 聊天区高度固定，消息超出时内部滚动

---

## 4. 业务规则与逻辑

- 整页为单页滚动结构，无跳转至其他页面
- 导航锚点点击后平滑滚动至对应区域
- 数字分身聊天区：用户输入为空时，禁止发送；发送后输入框自动清空
- 数字分身回复基于预设知识范围作答，不涉及范围外内容时可回复：「这个问题超出了我目前的知识范围，欢迎直接联系 Jesse 了解更多。」

---

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 用户输入为空点击发送 | 不触发发送，输入框轻微抖动提示 |
| 数字分身接口请求失败 | 显示提示文字：「分身暂时无法回复，请稍后再试」 |
| 数字分身问题超出知识范围 | 回复引导文案，建议直接联系 Jesse |
| 移动端窄屏下内容溢出 | 所有区块自适应单列布局，字号适配缩放 |

---

## 6. 验收标准

- 首页可正常加载，Hero 区、个人信息区、聊天区均完整展示
- 导航锚点点击后可平滑滚动至对应区域
- 数字分身聊天区可正常输入、发送、展示回复
- 空输入时发送无效，有视觉反馈
- 移动端（375px 宽度基准）布局正常，无内容溢出或错位
- 整体配色以紫色、深蓝色、白色为主，风格简约清爽

---

- 后台内容管理（头像、文案的动态编辑）
- 访客统计与数据分析

- 数字分身对话历史持久化存储
